# Codebase Structure

**Analysis Date:** 2026-03-01

## Directory Layout

```text
Crewtrace-landing/
|-- app/                # App Router pages, layouts, metadata endpoints
|-- components/         # Reusable UI and interactive conversion components
|   `-- __tests__/      # Component-level tests
|-- content/            # Markdown content source
|   `-- blog/           # Blog post markdown files
|-- lib/                # Structured data modules and SEO utilities
|-- public/             # Static assets (images, video, icons)
|-- docs/               # Brand/reference documentation
|-- .codex/             # GSD and agent workflow resources
|-- package.json        # Project scripts and dependencies
|-- next.config.ts      # Next.js configuration
|-- tsconfig.json       # TypeScript compiler configuration
|-- vitest.config.ts    # Vitest configuration
`-- eslint.config.mjs   # ESLint configuration
```

## Directory Purposes

**`app/`:**
- Purpose: Route definitions and page composition.
- Key patterns: `page.tsx`, `layout.tsx`, dynamic route folders like `[slug]`.
- Key files: `app/layout.tsx`, `app/page.tsx`, `app/sitemap.ts`, `app/robots.ts`.

**`components/`:**
- Purpose: Shared UI blocks and interactive lead tools.
- Key files: `Navbar.tsx`, `Footer.tsx`, `Hero.tsx`, `SavingsCalculator.tsx`, `ComplianceAuditTool.tsx`.
- Test location: `components/__tests__/ComplianceAuditTool.test.ts`.

**`content/blog/`:**
- Purpose: Markdown source for blog pages.
- Consumed by: `lib/blog.ts`.

**`lib/`:**
- Purpose: Data/content registries and SEO helper functions.
- Key files: `seo.ts`, `solutions.ts`, `guides.ts`, `caseStudies.ts`, `blog.ts`.

**`public/`:**
- Purpose: Static media served as-is.
- Contains: logos, hero image, og image, demo video, app icons.

## Key File Locations

**Entry points:**
- `app/layout.tsx` - global shell.
- `app/page.tsx` - homepage.

**SEO infra:**
- `lib/seo.ts` - metadata factory + site config.
- `app/sitemap.ts` - sitemap generation.
- `app/robots.ts` - robots rules.

**Dynamic content routes:**
- `app/features/[slug]/page.tsx`
- `app/industries/[slug]/page.tsx`
- `app/guides/[slug]/page.tsx`
- `app/case-studies/[slug]/page.tsx`
- `app/blog/[slug]/page.tsx`

**Interactive conversion pages:**
- `app/calculator/page.tsx`
- `app/compliance-audit/page.tsx`

## Naming Conventions

**Files:**
- Route files: Next conventions (`page.tsx`, `layout.tsx`).
- Components: PascalCase (`FeatureGrid.tsx`).
- Utility/data modules: camel/lowercase single-word files in `lib/`.
- Tests: `*.test.ts` under `components/__tests__/`.

**Directories:**
- Kebab-case for routes (`case-studies`, `compliance-audit`).
- Dynamic route segments in bracket form (`[slug]`).

## Where to Add New Code

**New SEO template page type:**
- Route: `app/<cluster>/[slug]/page.tsx`.
- Data source: add/update `lib/<cluster>.ts`.
- Include in sitemap: `app/sitemap.ts`.

**New reusable section/component:**
- Add to `components/`.
- Reuse across route pages to maintain consistency.

**New long-form content item:**
- Blog content to `content/blog/*.md`.
- Guide/case-study data to corresponding `lib/*.ts` arrays.

## Special Directories

**`.codex/`:**
- Purpose: Agent workflows and GSD command resources.
- Source: local tooling metadata.
- Committed: yes.

**`.next/`:**
- Purpose: Next.js build output.
- Source: generated by `next dev/build`.
- Committed: no.
